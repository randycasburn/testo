The other non-angular static page

What is different from a SPA?

No router based Component lazy-loading

No client side routing

Uses traditional full round-trip requests for each page.

Challenges?

The browser window is reloaded from scratch on each navigation (No SPA).

Navigating to a new page bootstraps a brand new angular app each time.

Consequences?

Data/state are not persisted between page changes.

You will have to persist data/state in localStorage, IndexDB, server side, etc.

Data/state management (moving data into and out of your components) is your responsibility.
